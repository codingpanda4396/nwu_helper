# H5 首页、后台路由与功能边界报告

## 线上当前问题确认

- `/` 和 `/admin` 由同一个前端包、同一个入口承载。修改前未按路径区分用户首页和平台后台，访问根路径会直接进入后台登录界面。
- 后台登录页曾在前端预填演示账号和密码，存在生产页面暴露弱口令示例的风险。
- Web 前端在未配置 `VITE_API_BASE` 时曾默认请求 `http://localhost:3000`，生产构建部署到线上域名后会把 API 请求打向访问者本机或无效地址。

## 修改后的功能边界

- 用户 H5：`/` 和 `/student` 渲染学生端 H5 首页，读取公开首页、随机餐饮推荐和商家详情接口；网络失败时展示空状态，不阻塞页面打开。
- 平台后台：`/admin` 和 `/admin/*` 渲染平台管理员登录与运营后台，继续使用管理员 token 访问 `/api/admin/*`。
- 公开 API：`/api/public/home`、`/api/public/food/random`、`/api/public/merchants/:id` 等接口面向用户 H5 公开读取。
- 管理员 API：`/api/admin/*` 仅供平台管理员后台使用，需要后台登录获得的 Bearer token。
- 商家 API / 商家后台：本次不新增完整商家 H5 后台；商家侧经营、核销、资料维护仍属于独立边界，后续应单独放在 `/merchant` 或等价入口。

## 已验证范围

- 前端路由边界：`/`、`/student` 进入用户 H5；`/admin` 进入平台后台登录/管理界面。
- 登录表单：账号和密码初始值为空，保留浏览器标准 `autocomplete`，不再写死任何账号密码。
- 公开数据读取：H5 首页调用 `/api/public/home`，随机推荐调用 `/api/public/food/random`，商家详情调用 `/api/public/merchants/:id`。
- API 默认地址：Web 前端未配置 `VITE_API_BASE` 时默认使用同源 `""`，本地开发继续通过 Vite `/api -> http://localhost:4000` 代理。
- 后台鉴权入口：管理员登录成功后继续把 token 和用户信息写入 `localStorage`，后台请求继续携带 Bearer token。
- 静态检查目标：`pnpm --filter @nwu-helper/web typecheck`、`pnpm --filter @nwu-helper/web build`、`pnpm --filter @nwu-helper/api test`。

## 未覆盖范围

- 完整商家后台 H5，包括商家登录、经营数据、资料维护、优惠券维护、核销操作。
- 在线支付、微信支付、自动分账、复杂交易结算和财务对账。
- 微信登录、手机号授权、实名学生认证和用户账号体系。
- 原生客户端发布流程、平台审核材料和生产 AppID 配置。
- 复杂内容审核、风控、反作弊、营销归因报表和多角色权限矩阵。

## 剩余风险

- 线上生效需要重新构建并发布 Web 前端产物，同时确认服务端对 `/admin`、`/student` 等 SPA 路径回退到前端入口。
- 真实数据为空时，H5 首页依赖空状态展示；运营侧需要配置轮播、活动、餐饮随机推荐和西大圈入口才能形成完整首页体验。
- 公开商家详情只展示现有接口字段，若优惠券状态、库存或领取流程需要更强交互，需要继续补用户侧领取链路。
