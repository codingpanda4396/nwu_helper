# ROLE

你是资深微信小程序产品经理 + 全栈架构师 + 本地生活产品专家。

请帮助我重构一个已经存在的uniapp项目。

核心要求：

这次不是简单 UI 修改，而是一次「产品定位升级 + 架构重构」。

目标是：

从：

“西北大学校园账号（西大圈）的小程序”

升级为：

“可复制到任意大学的校园本地生活增长平台”。

西大圈未来只是平台中的：

【第一个合作品牌 / 入驻客户 / 校园流量IP】

而不是平台本身。

---

# 一、核心产品定位

新产品定位：

一个服务大学生的：

【校园本地生活 × 商家增长平台】

连接：

大学生需求
⇅
校园周边商家

能力包括：

- 商家展示
- 优惠活动
- 优惠券领取
- 到店核销
- 社群引流
- 商家私域
- 数据归因
- 校园讨论区
- 校园信息服务

长期目标：

做成：

“校园版美团 + 校园版大众点评 + 校园增长服务商”

但 MVP 极简。

核心原则：

【先真实商业闭环，再做复杂功能】

不要过度设计。

---

# 二、重构目标

请重新设计整个产品信息架构（IA）与页面结构。

重点参考：

- 美团
- 抖音团购
- 小红书本地生活
- 大众点评
- 校园社群产品

要求：

界面必须更成熟、更统一、更像真实商业产品。

不能像学生作业。

必须具备：

“可信赖、年轻、有校园感、转化强”

的感觉。

目标：

强转化 + 校园感 + 活动运营。

模块：

- 校园切换（未来支持多个大学）

### Banner

- 校园抽奖
- 新店福利

---

### 核心金刚区（8宫格）

推荐：

- 夜宵

### 热门商家推荐

- 距离
- 优惠信息
